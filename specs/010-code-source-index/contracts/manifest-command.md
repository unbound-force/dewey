# Contract: Manifest CLI Command

**Package**: `main`  
**File**: `cli.go` (function `newManifestCmd()`)

## Command: `dewey manifest`

```
Usage: dewey manifest [flags]

Flags:
  --vault string   Path to vault (default: current directory)
```

## Behavior

1. Resolve vault path (CWD or `--vault` flag)
2. Walk `.go` files in the vault path using the ignore matcher
3. Run the Go chunker on each file (skip test files, skip syntax errors)
4. Collect all extracted blocks, categorized by `Kind`
5. Generate `.dewey/manifest.md` with sections:
   - Header with generation timestamp
   - CLI Commands section (from `Kind: "command"`)
   - MCP Tools section (from `Kind: "tool"`)
   - Exported Packages section (from `Kind: "package"`)
6. Create `.dewey/` directory if it doesn't exist
7. Write manifest file (overwrite if exists)
8. Log success with block counts

## Output Format

```markdown
# Project Manifest

> Auto-generated by `dewey manifest` on 2026-04-06T10:30:00Z
> Do not edit manually — regenerate with `dewey manifest`

## CLI Commands

| Command | Description |
|---------|-------------|
| serve | Start the MCP server |
| index | Build or update the knowledge graph index |
| doctor | Check Dewey prerequisites |

## MCP Tools

| Tool | Description |
|------|-------------|
| get_page | Get a page with its block tree |
| search | Full-text search across all blocks |
| semantic_search | Vector similarity search |

## Exported Packages

### package main

Knowledge graph MCP server & CLI

### package vault

Obsidian vault backend with file parsing and indexing

### package store

SQLite persistence layer for pages, blocks, links, embeddings
```

## Error Handling

- No `.go` files found: write minimal manifest with "No Go source files found" note
- All files have syntax errors: write manifest with "No declarations extracted" note
- `.dewey/` directory doesn't exist: create it
- File write error: return error to CLI

## Exit Codes

- 0: Manifest generated successfully
- 1: Error (file write failure, etc.)

## Invariants

1. Manifest MUST be a valid markdown file indexable by disk sources (FR-012)
2. Manifest MUST include generation timestamp
3. Manifest MUST use the same chunker as the code source (FR-011)
4. Manifest MUST NOT include test file content
5. Sections with no entries MUST be omitted (not shown as empty tables)
6. Command MUST be idempotent (running twice produces the same output for the same input)
